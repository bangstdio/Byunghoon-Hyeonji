## 4. 기술 스택
 
### 핵심 방향
- 별도 프레임워크 없음 — 순수 HTML / CSS / Vanilla JS
- 외부 의존성 최소화 → 로딩 속도 최적화 + 유지보수 단순화
- 바이브코딩 친화적 구조 — 파일 분리 / CSS 변수 기반 디자인 토큰
- 카운트다운 및 일정 등록 데이터는 JavaScript 상수로 관리
 
### 스택 상세
 
| 영역 | 기술 | 비고 |
|------|------|------|
| 마크업 | HTML5 | 단일 페이지 |
| 스타일 | CSS3 | 커스텀 프로퍼티로 디자인 토큰 관리 |
| 인터랙션 | Vanilla JS | UI 제어 / 카운트다운 / 팝업 |
| 스크롤 애니메이션 | GSAP + ScrollTrigger | CDN 로드 / 무료 플랜 |
| 폰트 | Google Fonts (Inter) | NotionInter 대체 |
| 일정 등록 | JavaScript Blob | 브라우저 단에서 `.ics` 동적 생성 및 다운로드 |
| 배포 | GitHub Pages / Netlify | 무료 / 정적 호스팅 |
 
### GSAP ScrollTrigger 선택 이유
 
| 인터랙션 | Vanilla JS | GSAP ScrollTrigger |
|----------|------------|--------------------|
| 콜라주 사진 날아들기 | 복잡한 좌표 수식 필요 | `from` / `to` 선언으로 처리 |
| 단어별 컬러 전환 | threshold 직접 계산 | `scrub` 옵션으로 자동 |
| 중앙 사진 스택 시퀀스 | 스크롤 제어 복잡 | `pin` / `scrub` / timeline으로 처리 |
| 다이내믹 아일랜드 | 가능하나 복잡 | 간결하게 처리 |

## 6. 상세 기능 명세
 
### F-01 / F-02 — 타이틀 콜라주 (SECTION 1)
 
**초기 상태**
- 메인 사진 `width: 100vw` / `height: 100vh` / `object-fit: cover`
- 타이틀 텍스트 화면 중앙 오버레이
  - 내용: `Byunghoon & Hyeonji`
  - 스타일: Inter 700 / 데스크톱 64px / 모바일 36px / `#ffffff`
 
**스크롤 시퀀스**
1. 메인 사진 `scale(1.0)` → `scale(0.6)` (중앙 기준 축소)
2. 보조 사진 5장 화면 사방 밖 → 지정 콜라주 좌표로 이동
3. 타이틀 텍스트 메인 사진과 동기화 축소 → `opacity: 0`
4. 콜라주 완성 상태에서 다음 섹션으로 자연스럽게 연결
 
**사진 배치**
- 메인 1장 + 보조 5장 = 총 6장
- 각 사진: `border-radius: 16px` / 회전 `±3~5deg`
- 보조 사진 크기 제각각 (자연스러운 콜라주)

### F-04 — 우리의 이야기 스크롤 시퀀스 (SECTION 3)
 
#### 공통 구조
- 스티키 타이틀 `우리의 사랑 이야기`가 상단에 고정
- 사진은 중앙에서 한 장씩 올라와 stack
- 다음 사진은 이전 사진 위로 겹쳐짐
- 텍스트는 사진과 분리되어 등장하며 순서에 따라 좌우 교차 또는 정렬감 교차를 적용
- 총 6~7장 사진 시퀀스 반복
 
#### 스티키 타이틀
- 내용: `우리의 사랑 이야기`
- 데스크톱: `48~64px` / 700 / 큰 스티키 헤더형
- 모바일: `28~32px` / 700 / 축소 스티키 헤더형
- `position: sticky` / 다이내믹 아일랜드 하단에 고정
- `z-index: 1` 로 설정
- 사진 레이어: `z-index: 10` 이상
- 텍스트 레이어: `z-index: 5~15` 범위에서 조정
 
#### 다이내믹 아일랜드 및 sticky 기준값
- 다이내믹 아일랜드 높이: 모바일 기준 약 `48~56px`
- SECTION 3 sticky 요소의 `top` 값: safe area와 네비게이션 높이를 고려해 약 `88~104px` 범위에서 조정
 
#### 데스크톱 동작
```text
① 큰 스티키 타이틀 고정
② 중앙에 사진 1장 올라와 멈춤
③ 설명 텍스트가 사진 바깥 좌측 또는 우측에 번갈아 등장
④ 더 스크롤하면 텍스트 fade-out
⑤ 다음 사진이 중앙에서 이전 사진 위로 겹쳐짐
⑥ 반대편 방향의 텍스트 등장
⑦ 반복
```
- 사진은 중앙 스택 구조 유지
- 텍스트는 사진 바깥 좌/우에 번갈아 배치
- 에디토리얼 레이아웃 느낌으로 구성
 
#### 모바일 동작
```text
① 축소된 스티키 타이틀 고정
② 사진이 타이틀 바로 아래 중앙에 올라와 고정
③ 더 스크롤하면 설명 텍스트가 사진 아래에서 등장
④ 홀수 텍스트는 좌정렬, 짝수 텍스트는 우정렬
⑤ 더 스크롤하면 텍스트가 위로 흘러 올라가며 opacity 감소로 사라짐
⑥ 다음 사진이 이전 사진 위에 겹쳐지며 올라옴
⑦ 다음 텍스트 등장
⑧ 반복
```
- 모바일 사진 높이: 약 `38~42vh`
- 모바일 텍스트 폭: 약 `88%`
- 텍스트 소멸 방식: `opacity` 감소 + `translateY(-24px)` (위로 흘러 사라짐)
- 사진 스택 간 시각적 중첩 간격: 약 `16~24px`
 
**사진 스펙**
- `border-radius: 16px`
- `1px solid rgba(0,0,0,0.1)`
- `var(--shadow-card)` 적용
 
**텍스트 스펙**
- 제목: Inter 700 / 22px / `rgba(0,0,0,0.95)`
- 설명: Inter 400 / 16px / `#615d59`

### F-06 / F-07 — 정보 카드 + 바텀시트 팝업 (SECTION 5)
 
**카드 구성**
 
| # | 제목 | 팝업 내용 |
|---|------|-----------|
| 1 | 오시는 길 | 지도 + 주소 + 교통편 |
| 2 | 마음 전하실 곳 | 계좌번호 (신랑/신부 측) |
| 3 | 식사 안내 | 식사 장소 및 안내 문구 |
| 4 | 주차 안내 | 주차 방법 및 안내 문구 |
| 5 | 방명록 | placeholder 팝업 우선 구성 / 추후 저장 기능 연동 가능 |
| 6 | 결혼식 컨셉 | 웨딩 컨셉 소개 텍스트 + 사진 |
 
**카드 스펙**
- 비율: 3:4
- 모바일: 2열 그리드
- 데스크톱: 3열 그리드
- `border-radius: 16px`
- 배경: 사진 + 하단 그라디언트 오버레이 (`rgba(0,0,0,0)` → `rgba(0,0,0,0.5)`)
- 제목: Inter 700 / 좌측 상단 / `#ffffff`
 
**바텀시트 스펙**
- 트리거: 카드 클릭
- 애니메이션: 하단에서 슬라이드 업 (`transform: translateY`)
- 형태: `border-radius: 16px 16px 0 0` / 배경 `#ffffff`
- 닫기: 상단 드래그 핸들 또는 우측 X 버튼

## 8. 제약사항
 
### 기술적 제약
- 백엔드 없음 → 방명록 저장 기능 / RSVP는 외부 서비스 연동 필요 (Google Sheets 등)
- GSAP ScrollTrigger는 CDN 로드 방식 사용 (오프라인 환경 미지원)
- 일정 정보 변경 시 JavaScript 내 이벤트 변수(`WEDDING_DATE` 등)를 업데이트해야 함
 
### 콘텐츠 제약
- 사진 미수급 → 초기 개발 시 `https://picsum.photos` 등의 더미 이미지 URL 사용 후 추후 교체
- 필요 사진 수: 총 `18~19장`
  - 타이틀 콜라주: 6장 (메인 1 + 보조 5)
  - 우리의 이야기: 6~7장
  - 자세한 정보 카드 배경: 6장
- 방명록 상세 구현 방식은 초기 버전에서 placeholder 팝업 기준으로 설계
 
### 브라우저 제약
- 타깃: 모바일 Safari / Chrome 최신 버전 우선
- `backdrop-filter` (블러)는 구형 Android 브라우저에서 미지원 가능성 있음 → fallback 처리 필요
- IE 미지원
 
### 배포 제약
- 커스텀 도메인 미사용 시 GitHub Pages 기본 URL 제공
- 커스텀 도메인 사용 시 별도 도메인 구매 필요
